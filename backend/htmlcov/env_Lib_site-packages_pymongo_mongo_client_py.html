81 """ 

82 A client-side representation of a MongoDB cluster. 

83 

84 Instances can represent either a standalone MongoDB server, a replica 

85 set, or a sharded cluster. Instances of this class are responsible for

105 """Client for a MongoDB instance, a replica set, or a set of mongoses. 

106 

107 The client object is thread-safe and has connection-pooling built in. 

108 If an operation fails because of a network error, 

109 :class:`~pymongo.errors.ConnectionFailure` is raised and the client 

110 reconnects in the background. Application code should handle this 

111 exception (recognizing that the operation failed) and then continue to 

112 execute. 

113 

114 The `host` parameter can be a full `mongodb URI 

115 <http://dochub.mongodb.org/core/connections>`_, in addition to 

116 a simple hostname. It can also be a list of hostnames or 

117 URIs. Any port specified in the host string(s) will override 

118 the `port` parameter. If multiple mongodb URIs containing 

119 database or auth information are passed, the last database, 

120 username, and password present will be used. For username and 

121 passwords reserved characters like ':', '/', '+' and '@' must be 

122 percent encoded following RFC 2396:: 

123 

124 try: 

125 # Python 3.x 

126 from urllib.parse import quote_plus 

127 except ImportError: 

128 # Python 2.x 

129 from urllib import quote_plus 

130 

131 uri = "mongodb://%s:%s@%s" % ( 

132 quote_plus(user), quote_plus(password), host) 

133 client = MongoClient(uri) 

134 

135 Unix domain sockets are also supported. The socket path must be percent 

136 encoded in the URI:: 

137 

138 uri = "mongodb://%s:%s@%s" % ( 

139 quote_plus(user), quote_plus(password), quote_plus(socket_path)) 

140 client = MongoClient(uri) 

141 

142 But not when passed as a simple hostname:: 

143 

144 client = MongoClient('/tmp/mongodb-27017.sock') 

145 

146 Starting with version 3.6, PyMongo supports mongodb+srv:// URIs. The 

147 URI must include one, and only one, hostname. The hostname will be 

148 resolved to one or more DNS `SRV records 

149 <https://en.wikipedia.org/wiki/SRV_record>`_ which will be used 

150 as the seed list for connecting to the MongoDB deployment. When using 

151 SRV URIs, the `authSource` and `replicaSet` configuration options can 

152 be specified using `TXT records 

153 <https://en.wikipedia.org/wiki/TXT_record>`_. See the 

154 `Initial DNS Seedlist Discovery spec 

155 <https://github.com/mongodb/specifications/blob/master/source/ 

156 initial-dns-seedlist-discovery/initial-dns-seedlist-discovery.rst>`_ 

157 for more details. Note that the use of SRV URIs implicitly enables 

158 TLS support. Pass tls=false in the URI to override. 

159 

160 .. note:: MongoClient creation will block waiting for answers from 

161 DNS when mongodb+srv:// URIs are used. 

162 

163 .. note:: Starting with version 3.0 the :class:`MongoClient` 

164 constructor no longer blocks while connecting to the server or 

165 servers, and it no longer raises 

166 :class:`~pymongo.errors.ConnectionFailure` if they are 

167 unavailable, nor :class:`~pymongo.errors.ConfigurationError` 

168 if the user's credentials are wrong. Instead, the constructor 

169 returns immediately and launches the connection process on 

170 background threads. You can check if the server is available 

171 like this:: 

172 

173 from pymongo.errors import ConnectionFailure 

174 client = MongoClient() 

175 try: 

176 # The ismaster command is cheap and does not require auth. 

177 client.admin.command('ismaster') 

178 except ConnectionFailure: 

179 print("Server not available")

769 if credentials == all_credentials[source]:

863 """Watch changes on this cluster. 

864 

865 Performs an aggregation with an implicit initial ``$changeStream`` 

866 stage and returns a 

867 :class:`~pymongo.change_stream.ClusterChangeStream` cursor which 

868 iterates over changes on all databases on this cluster. 

869 

870 Introduced in MongoDB 4.0. 

871 

872 .. code-block:: python 

873 

874 with client.watch() as stream: 

875 for change in stream: 

876 print(change) 

877

884 change events are missed. Any error encountered during the resume 

885 attempt indicates there may be an outage and will be raised. 

886 

887 .. code-block:: python 

888 

889 try: 

890 with client.watch( 

891 [{'$match': {'operationType': 'insert'}}]) as stream: 

892 for insert_change in stream: 

893 print(insert_change) 

894 except pymongo.errors.PyMongoError: 

895 # The ChangeStream encountered an unrecoverable error or the 

896 # resume attempt failed to recreate the cursor. 

897 logging.error('...') 

898 

899 For a precise description of the resume process see the 

900 `change streams specification`_. 

901 

902 :Parameters: 

903 - `pipeline` (optional): A list of aggregation pipeline stages to

911 document, as well as a copy of the entire document that was 

912 changed from some time after the change occurred.

914 change stream will start returning changes that occur directly 

915 after the operation specified in the resume token. A resume token

917 - `max_await_time_ms` (optional): The maximum time in milliseconds 

918 for the server to wait for changes before responding to a getMore 

919 operation. 

920 - `batch_size` (optional): The maximum number of documents to return 

921 per batch. 

922 - `collation` (optional): The :class:`~pymongo.collation.Collation` 

923 to use for the aggregation. 

924 - `start_at_operation_time` (optional): If provided, the resulting 

925 change stream will only return changes that occurred at or after 

926 the specified :class:`~bson.timestamp.Timestamp`. Requires 

927 MongoDB >= 4.0. 

928 - `session` (optional): a 

929 :class:`~pymongo.client_session.ClientSession`. 

930 - `start_after` (optional): The same as `resume_after` except that 

931 `start_after` can resume notifications after an invalidate event. 

932 This option and `resume_after` are mutually exclusive. 

933 

934 :Returns:

946 """

949 batch_size, collation, start_at_operation_time, session,
